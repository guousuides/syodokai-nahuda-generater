# 名札PDF生成ツール (`nahuda.exe` / `nahuda.py`)

早稲田書道會の73期の郭です
活動で使用する名札を作るツールをここにまとめます
指定された「参加者データ（CSVまたは手動入力）」と「背景画像（PNG）」を組み合わせて、A4サイズの用紙に10人分（2列×5行）の名札用PDFファイルを自動生成・出力するアプリケーションです。

## 1. 機能概要

- **CSV読み込み機能**: 大量データを一気にPDFへ変換可能です。
- **手動入力機能**: アプリ上で少人数の名札を追加作成し、PDFへ変換できます。
- **背景合成機能**: お好きなA4比率の背景画像(PNG)を指定して合成できます。
- **フォント自動適用**: 専用の太文字ポップ体フォントで名前や役職などを描画します。

## 2. 動作環境・前提条件

本ツール（およびソースコード）は以下の環境で動作を想定しています。
- **対象OS**: Windows OS
- **必須フォント**: `HG創英角ﾎﾟｯﾌﾟ体` (`C:/Windows/Fonts/HGRKK.TTC`)
  ※Windowsに標準搭載されている場合が多いですが、無い場合はレイアウト崩れやエラーになる可能性があります。

## 3. 使い方：実行ツール (`nahuda.exe`) 編

`nahuda.exe` を用意している場合、ダブルクリックするだけで起動できます。
起動すると「**名札PDF生成ツール**」という小さな画面が開き、2つのモードを選択できます。

### パターンA: CSVファイルからの一括作成

大量の参加者名簿がある場合（例: `名札 .csv`）はこちらを使用します。

**【手順】**
1. メイン画面で「**CSVファイルから読み込み**」ボタンをクリックします。
2. 名札のデータが記載された **CSVファイル** を選択します。
3. 名札の **背景画像 (PNGファイル)** を選択します。（※A4縦の比率を推奨）
4. 出力される **PDFの保存先・ファイル名** を指定して「保存」を押します。
5. 「PDF生成が完了しました」というメッセージが出れば成功です。

**【CSVファイルの書き方ルール】**
1行目（ヘッダー行）は以下の列名に完全一致している必要があります。
存在しない列は空欄として無視されます。
- `名前` **（必須）**: 参加者の氏名
- `フリガナ` : 小さい文字で名前の上に表示されます
- `学部学年` : 対象者の所属や学年など（右端に表示）
- `役職` : 肩書き・役職その1（左下に表示）
- `役職2` : 肩書き・役職その2（役職の下に表示）

### パターンB: 手動入力での作成

数人だけ追加で作成したい場合や、手元にCSVがない時に使用します。

**【手順】**
1. メイン画面で「**手動入力**」ボタンをクリックします。別ウィンドウが開きます。
2. 上部の入力欄に「名前」「フリガナ」「学部学年」「役職」「役職2」を入力します。
   （「名前」は必須項目です）
3. 「**追加**」ボタンを押すと、下部のリスト空間にそのデータが記録されます。
   ※間違えた場合は、リスト内の対象行をクリックして選択し、「**削除**」ボタンを押します。
4. 全員の入力を終えたら、右下の「**PDF生成**」ボタンを押します。
5. 操作はCSVモードと同様に、**背景画像(PNG)** → **保存先(PDF)** の順で選択すれば完了です。

## 4. PDF出力の仕様一覧

出力されたPDFの詳細なレイアウト仕様は以下の通りです。
- **用紙サイズ**: A4サイズ（縦位置 / Portrait）
- **割り付け**: 1ページあたり **横2列 × 縦5行 = 計10人分**
- **出力フォント**: すべて `HG創英角ﾎﾟｯﾌﾟ体` (HGRKK)
- **テキストサイズと配置**:
  - `名前`: 中央配置 (サイズ 26)
  - `フリガナ`: 中央配置 (サイズ 10 / 名前の直上)
  - `学部学年`: 右寄せ (サイズ 13 / 右下付近)
  - `役職`: 左寄せ (サイズ 12 / 左下付近)
  - `役職2`: 左寄せ (サイズ 12 / 役職のさらに下部)

## 5. 使い方：ソースコード (`nahuda.py`)・開発者向け

Python環境がセットアップされたPC上で直接スクリプトを実行することも可能です。

**【依存パッケージのインストール】**
```bash
pip install pandas reportlab
```
※標準ライブラリの `tkinter` を使用しています。

**【スクリプトの実行】**
対象ディレクトリにて以下を実行します。
```bash
python nahuda.py
```

## 6. メンテナンス・カスタマイズの方法

`nahuda.py` 内のソースコードを編集することで、細かい設定値の調整が可能です。
（※編集後は再度EXE化する必要があります）

### (1) レイアウトや座標（印字位置）の調整
`create_pdf(data, img_path, save_path)` 関数内にて座標をセットしています。（A4の左下が基準座標 0,0）
- 全体の間隔調整: `offset_x` (横の間隔), `offset_y` (縦の間隔)
- 個別の位置調整: `base_x` (名前のX座標), `base_y` (名前のY座標), `year_y`, `pos_y` など。
- 役職2の高さ調整: `base_y_2_val = 270 * mm` の値を上下させることで変更可能。

### (2) フォント種類の変更・サイズ調整
- 別のフォントにする場合: `pdfmetrics.registerFont(TTFont("HGRKK", "C:/Windows/Fonts/..."))` のパスを指定。
- フォントサイズの変更: `page.setFont("HGRKK", 26)` 等の `26` 部分の数値を編集してください。

### (3) 新しい入力項目を追加する場合
CSVヘッダーやGUI手動画面に別の項目（例: 「ふりがな2」等）を追加する場合は、次の3箇所をコード上で同時に追加する必要があります。
1. GUIフォームへの `tk.Label`・`tk.Entry` の追加
2. リスト表示 (`Treeview`) のカラム設定の追加
3. `add_entry()`, `generate_pdf()` メソッド内の辞書データのキー (`data_list.append(...)`) への追加

### (4) EXEファイル化（ビルド手順）
ソースコード変更後、実行ファイルを書き出すための `PyInstaller` コマンドです。
```bash
pip install pyinstaller
pyinstaller --onefile --windowed nahuda.py
```
> ※ 実行後、`dist` フォルダ内に 新しい `nahuda.exe` が生成されます。

## 7. よくある質問・トラブルシューティング

- **Q. 「CSVの読み込みに失敗しました」というエラーが出る**
  - **A.** CSVファイルの文字コード（エンコーディング）が原因である可能性が高いです。ファイルをExcelで開き、「名前を付けて保存」から `CSV UTF-8` または標準の `CSV (コンマ区切り)` 形式で保存し直してください。

- **Q. 背景画像に指定できない**
  - **A.** 選択可能な画像は `PNG` 形式（`.png`）のみです。JPEG等を使用したい場合は画像の変換を行うか、プログラム側の `filetypes` に `*.jpg` 等の拡張子を許可する改修が必要です。

- **Q. 文字がはみ出してしまう**
  - **A.** 長い肩書きや名前が想定内枠を超える場合があります。長い場合は適宜スペースを入れるか、上記「メンテナンス」の項目に沿ってフォントサイズを下げた版(`nahuda.py`)で運用してください。

- **Q. 手動入力ウィンドウを間違えて閉じてしまった**
  - **A.** メイン画面の「手動入力」ボタンを再度押すことで新しく開きますが、**前に入力していたリストの内容はリセット**されるためご注意ください。
